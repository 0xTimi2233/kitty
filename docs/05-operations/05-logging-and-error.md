# Logging 与 Error

日志使用统一事件模型：

```text
timestamp level event msg fields
```

要求：

- event 是稳定枚举值。
- msg 使用 Display 文本。
- fields 平铺输出。
- 错误消息和日志消息使用英文。
- 注释和编排说明使用中文。
- 反序列化错误将带路径的 display 文本放入 msg。
- 非反序列化阶段需要维护可定位的配置路径。
- 代码不得输出敏感日志。

## 敏感日志原则

本阶段 docs 只记录原则：开发代码时约定不输出敏感日志。

本文件不新增敏感字段清单、错误摘要字段清单或 debug dump 边界。后续实现阶段如果需要更严格验收，可以重新从 `docs/` 立项并补充正式需求。

## Error 可定位性

配置相关错误必须尽量携带 config path。无法定位到具体字段时，至少携带所属阶段和配置对象路径。

日志与错误不得把当前 preflight 结论表述为正式需求冻结。
